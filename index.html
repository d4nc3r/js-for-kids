Introduction

JavaScript is a programming language used in web browsers, like Chrome and Firefox. It's a way to provide the browser with some instructions to follow - like changing the color of a button every time we click it. Almost every website uses at least a little bit of JavaScript, and all of the "blocks" you used in yesterday's exercises used JavaScript.

Try clicking the button, and see what happens.

Now right-click on the button and choose "inspect" - and notice how we can see the HTML code for our button now. Try clicking the button again. Pay attention to how the background-color style in the HTML changes - that's JavaScript doing that! Let's learn a little bit about how JavaScript works so we can write our own code. (for later - check out some HTML color names)

We don't need any special tools to try writing JavaScript code because it runs right in our browser! Look for the "Console" tab in your Developer Tools. If you accidentally closed the Dev Tools, hit F12 or right-click anywhere on the web page and choose "Inspect" again.

We can type our JavaScript in this console, and any time we hit the enter key the browser will run it for us! Try typing a math equation, like 3 + 5 and hitting enter to see the answer.

We can write equations with text (called strings in JavaScript) too. Any time we want to use a string, we need to use either single ' or double " quotes around the text, so javascript knows how to read it. Try typing a string of text into the console, and hit enter. If you forget to put the quotes around your string, JavaScript with display an error - try that out so you can see what it looks like.

We can even add strings together - try typing 'hello' + ' ' + 'world!' in the console and hit enter.

Variables & Equations

We can also store the answer to an equation in a variable so we can use it again later. To do that, we just add let at the beginning, give JavaScript a name for our variable, and set it equal to our equestion using an = sign: 

let answer = 3 + 5;
You may notice that you don't see the answer when you hit enter this time - that's because JavaScript put the answer inside our variable - to see the result we just need to type our variable name "answer", and hit enter!

We can store other kinds of values in variables, not just numbers. This time let's try storing some text. Create a variable called "greeting", set it equal to some text: 

let greeting = "hello world!";
Now let's say we want to combine some text with a variable, maybe we want to store a person's name and then use their name in our greeting. There's a special kind of quote called a backtick we can use to do that. When we put backticks ` around our text, we can include our variables inside by using $ and putting our variable inside of some curly braces. Let's create a new variable called name, and use that inside of a larger string called hello: 
let name = "jamie";
let hello = `hi, ${name}`;
Now when we type "hello" and hit enter - you should see "hi" and the value in your "name" variable!

Console.log and Alert

JavaScript has lots of built-in functionality. Right now, we're just going to look at a couple of functions that are really helpful for learning. First, let's try using alert to pop up a little message for us: 

alert("hello!");
Whatever you put between the parenthases ( and ) will be displayed in the pop up! You can even put a variable in between the parenthases, rather than putting your literal string or number. For example, remember we had our greeting variable from earlier? We can use that inside of the alert: 
alert(greeting);

Sometimes we want to print or log some information or a variable without popping up a new windows. In these cases, we can use console.log(). Just like with alert, the value in between the parenthases is what will be logged - but this time instead of in the little pop up window, our value will be printed out to the console screen. For example: 

console.log(greeting);
This is especially helpful when we want to see what our variable values are while our code is running.

Arrays

Now that we know how to create variables and how to log to the console, let's try something a little harder. Sometimes we need a list of values, and in JavaScript we call that an array. We can create an array by putting square brackets, [ and ], around all the values we want in our list, and commas in between the values.

For example, let's create an array of animals (you can pick your favorites): 

let animals = ["dog", "cat", "capybara"];
Now if we type our variable name "animals" in the console and hit enter - we see our list of values. Try clicking the little arrow on the left-hand side, it will open up our array and show each value on its own line, as well as how many items are in the list.

Indexes

JavaScript numbers every value in our array with an index so that we can get individual values from it. To get a single value out of our array, we type our variable name again, but this time we add square brackets with the index of the value we want in between them. If you look back at the array in your console (opened up so each item is on its own line), you should see numbers to the left of each value - those are the indexes!

Let's try getting the value of our first animal out of our array: 

animals[0];
It may look strange to use 0 instead of 1 to get the first item in our list, but JavaScript always starts counting at zero, so our first item is index zero.

Try practicing by asking JavaScript to display the 2nd or 3rd item in your list.

Length

If you need to know how many items are in an array, you can ask JavaScript the length, like this: 

animals.length;

Updating

Adding new values into our array is easy, we can use push to "push" a new value onto the end of our animals array: 

animals.push("giraffe");
Check the console again to see what your array looks like now!

Now let's say we want to remove the last item in our array - we can pop that value right off the end and store it in a variable, like this: 

let last = animals.pop();
Check the value of animals again, and now it has only 3 items again - giraffe is gone, but you can now find it in the variable called last.

We can also change a value in our array if we need to. Let's use what we know already to change the value of "cat" to be "kitten". After you run your code, try testing it by displaying animals one more time - you should no longer see cat, instead you'll see kitten :)

Objects

When we want to group some information together, we can use an object in JavaScript. For example, we may want data that represents a person. It might have a name, an age, and a favoriteColor: 

let person = {
  name: "Moana",
  age: "16",
  favoriteColor: "ocean blue"
}
Try copying and pasting the code into your Dev Tools console, hit enter, and type "person" to ask JavaScript to display our person object.

It looks a little bit like our array doesn't it, except with curly brackets instead of square ones. Try clicking the arrow on the left-hand side to open up the object. Just like with our array, it shows us each value on a separate line, but this time instead of an index we see our key on the left of our values.

If we can get a single value out of our array by saying arrayName[index], try to guess how you could get a single value from our object. (show answer)

We can use the key next to the value you want inside square brackets to get a single value: 

person["favoriteColor"];
There's also a special way to get values from an object called dot notation, a fancy way of saying add a dot and then the key for the value you want: 
person.name;

Now try writing our greeting from earlier, but use our person's name inside it. Remember, when we want to use a variable inside of our text, use the special backticks ` instead of quotes, and put ${} around the variable we want to use.

It might be tempting to put person inside the curly brackets - just to see what would happen, try using our person object inside our string of text, we can try it out using `console.log`: 

console.log(`hi, ${person}`};
That looks super weird, doesn't it? That's because an object isn't just one value, like person.name or person.age is, it's a group of values, so when JavaScript tries to put the whole thing into a string, it will just show us that it's an object by printing [object Object].

We just want to say "hi" to our person's name - so we can either use square brackets like this: 

let greeting = `hi, ${person["name"]}`;
or we can use dot notation instead, like this: 
let greeting = `hi, ${person.name}`;

Conditionals - If / Else

When we want the computer to perform a task *sometimes*, under certain conditions, that's called a conditional, or an "if/else" statement.

Let's start with an object to represent a pet: 

let pet = {
  name: "Fido",
  age: 2,
  type: "dog"
};
Now let's write a conditional statement to pop up an alert message that says "good dog!" if our pet is a dog: 
if (pet.type === "dog") {
  alert("good dog!");
}
Now what if our pet was actually a cat? Try changing your pet's type to be "cat", and run your if statement again.

Nothing happened! Does that mean our code is broken? Nope! Our if statement is checking to see if our pet's type is "dog", and it only pops up our alert if it is. Now that our pet's type is "cat", it no longer satisfies the condition of our statement.

If we want to say something else when our pet is not a dog, we can add to our conditional: 

if (pet.type === "dog") {
  alert("good dog!");
} else {
  alert("here kitty!");
}
Now try running your if/else statement when your pet's type is "cat" and again when its type is "dog" and see what happens.

Lastly - if you only want to pop an alert message if your pet is a cat or a dog, but do nothing if it's a different type of pet, you can add an else if instead of an else, like this: 

if (pet.type === "dog") {
  alert("good dog!");
} else if (pet.type === "cat") {
  alert("here kitty!");
}
Now if you set your pet's type to "turtle", you won't see an alert pop up at all!

Functions

Functions are like blocks, they group instructions together so we can run it more than once, sometimes with different values passed in to change the behavior of our function.

Some functions just DO something, for example - we can write a function to "sayHi", like this: 

function sayHi() {
  console.log("hello world!");
}
Now if we type "sayHi" and hit enter, javascript shows us exactly what instructions our function will run. If we want javascript to actually run them, we have to type sayHi() with the parenthases, this tells javascript to actually run our instructions for us, and we can do that as many times as we want.

We can write a more useful function by sending some data, called a parameter, into our function to be used by our instructions. Let's re-write our sayHi function, this time passing it a name: 

function sayHi(name) {
  console.log(`hi, ${name}!`);
}
Now we can call the function with any name we want inside the parenthases, and javascript will print out hi, and the name (try a few).

If we forget to put some text inside the parenthases, javascript prints out hi, undefined - and what undefined means it that javascript doesn't know the value of "name", because we forgot to give it a value. Any time you see undefined in the console, it probably means javascript was looking for a variable that doesn't have a value yet.

Now let's try a function that returns something. How about a function that takes two numbers, multiplies them, and returns the answer? It's almost like the functions we've written before, but this time we had a return statement. 

function times(num1, num2) {
  return num1 * num2;
}
Now you can call times with any 2 numbers you like, and it will return the value of the first number times the second number. This also means you can set a new variable equal to the return value of our function if we'd like to store the answer, like this: 
let answer = times(7, 15);
This time when we run the code it doesn't display the answer - because it was saved in our answer variable. Try displaying our answer variable in the console - and you should see what 7 times 15 is!

Loops

Sometimes we want to loop through some data or repeat our instructions more than once. There's a few different ways to do that in JavaScript, we're going to focus on the for loop and on the forEach function.

For Loop

We use a for loop any time we want to repeat instructions a certain number of times. First we set up how many times we'd like to repeat our code: 

for(let count = 0; count < 5; count++) {

}
This will create a variable called "count" just for our for loop and set it equal to zero to start. JavaScript will continue to loop as long as count is less than 5, and it will add 1 to count every time the code runs (that is what count++ does).

Second, we need to add some code in between the curly brackets - whatever we put there will run over and over, until count is no longer less than 5. For example, this will print out the value of count each time the loop runs. 

for(let count = 0; count < 5; count++) {
  console.log(count);
}
We could also use a for loop to print out all of our values in an array. Let's try creating a new array, with whatever values you'd like in it, for example: 
let foods = ["pizza", "cheeseburgers", "skittles"];
Now we'll create a for loop that will start with zero, and loop as long as the index is less than the length of the array: 
for(let index = 0; index < foods.length; index++) {
  console.log(foods[index]);
}

ForEach Loop

When we want to loop through all the values in an array, there's an easier way to do that than the for loop! Arrays have a special function called forEach that lets us loop through each value once: 

foods.forEach(food => {
  console.log(food);
})
